# A simple REST based CRUD demo project using Spring Boot and MongoDB
This project provides 3 REST APIs to work with a single collection (employees) in MongoDB.
 - GET employees list with and without filter and with pagination
 - GET employee by id
 - POST (create) new employee

## Requirements

For building and running the application you need:

- [Java 17](https://jdk.java.net/17/)
- [Maven 3](https://maven.apache.org)
- [MongoDB](https://www.mongodb.com/try/download/community)
- [Bruno](https://www.usebruno.com/downloads) or Postman or similar REST Client

## Running the application locally
First in MongoDB create a database with name `demo` and `employees` collection. If you want to use a different database then update `application.properties` file.

There are several ways to run a Spring Boot application on your local machine. One way is to execute the `main` method in the `net.akvk.demo.DemoApp` class from your IDE. You can also use following command to start the service on port 11080 with `/employee` context path.

```shell
java -Dserver.port=11080 -Dserver.servlet.context-path=/employee -jar target\sb-mongodb.jar
```

## Testing the GET employees API without filter
In REST client enter the following URL to get a paginated list of employees

<http://localhost:11080/employee/api>

## Testing the GET employees API with filter
In REST client enter the following URL for a generic query filter which searches a few string fields. Note the use of page and size for pagination. Based on number of documents in the collection you can specify the page and size. The maximum size is limited to 100.

<http://localhost:11080/employee/api?query=Chicago&page=1&size=5>

In REST client enter the following URL to filter by employee city and female employees

<http://localhost:11080/employee/api?city=Chicago&gender=F>

In REST client enter following URL to find all employees who joined in 2024

<http://localhost:11080/employee/api?beg_dateOfJoining=2024-01-01T00:00:00&end_dateOfJoining=2024-12-31T23:59:59>



## Additional Exercises
Following APIs have been left unimplemented as exercise for developers. Please clone the project and add APIs below
- PUT API to edit existing user. Add a salary field (Long) to Employee class and try to update from REST Client
- Use the salary field filter with min_salary and max_salary and verify the filter works
- DELETE API to delete an employee by Id
- Add unit tests for the new APIs and check the code coverage report generated by jacoco plugin.
